Voici quelque chose qu'il m'arrive assez souvent de faire, notamment pour mettre à jour ou synchroniser différentes machines. J'oublie la commande régulièrement. Admettons que vous ayiez plusieurs branches sur le remote, et que vous souhaitiez les rappatrier dans le dépôt local :

p=. !http://gitready.com/images/branches.png!

Les infos sur le remote sont accessibles de la façon suivante :

<pre>
$ git remote show origin
  * remote origin
    URL: *************
    Remote branch merged with 'git pull' 
      while on branch master
        master
      Tracked remote branches
        haml master
</pre>

La commande pour réaliser la synchro des branches disponible sur le remote est en fait plutôt simple :

<pre>
git checkout --track -b 
  <local branch> <remote>/<tracked branch>
</pre>

Pour mon exemple, nous pourrions donc faire :

@git checkout --track -b haml origin/haml@

Il existe en fait une syntaxe plus concise :

@git checkout -t origin/haml@
